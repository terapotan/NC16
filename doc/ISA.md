# 表記法
1. [10:5]オペランド　上位10ビットから上位5ビットはオペランドを表します**最下位ビットを0ケタ目としてビット数を数えていることに注意してください。**ただし単にnビットのデータがあると言ったときには、n桁のビットが存在することを表すものとします。この表記法はLogisim-evolution上のビット数の表記に由来しています。
2. [20:10]0b0000000011　上位20ビットから上位10ビットまでの値は2進数で0000000011です。
3. [0] 0b1　上位0ビット目の値は2進数で1です。
4. 禁止　この値が真理値表の出力欄/意味欄に記載されていた場合、その出力に対応する入力はしてはなりません。禁止出力に対応する入力が行われた場合の動作は未定義です。未定義または未定義動作は、どのように動作するか仕様で一切規定していないことを表し、この場合NC-16はどのような動作をしても仕様上許容されます。
5. X　Don't Careを表します。
6. 真理値表において他の行と同入力で、出力値が異なる場合、先に書かれた行の出力値を優先するものとします。例えば以下の真理値表では1行目と2行目の記載が衝突しますが、1行目の記載が優先されるため、入力1011に対する出力は1となります。

|bit4|bit3|bit2|bit1|出力|
|:--:|:--:|:--:|:--:|:--:|
|1|0|1|1|1|
|X|X|X|X|0|
# NC-16命令セット（Instruction Set Architecture）
## 概要
　NC-16の命令のうち命令長が8バイトのものをシングル命令と言います。複数のシングル命令をまとめて一つの命令としたものをマルチ命令と言います。例えばadd命令はシングル命令ですが、call命令はマルチ命令です。
　シングル命令の前半4バイトがオペコード、後半4バイトがオペランドを表します。オペランドを使用しないオペコードの場合、オペランドの値は0で埋めておく必要があります。
　シングル命令のビット構成は次の通りです。
1. [32:17]オペコード
2. [16:1]オペランド

　オペコードはさらに以下のビット構造に分解されます。
1. [32:25]ファンクションコード
2. [24:21]FuncInput1
3. [20:17]FuncInput2

　ファンクションコードはオペコードの中核となる部分であり、機械語がCPUに及ぼす動作を決定します。

　FuncInput1,FuncInput2はファンクションコードの対象となる要素を決定します。対象はレジスタ、及びメモリです。各対象に対応するFuncInput1,FuncInput2は次の通りです。

FuncInput1
|bit24|bit23|bit22|bit21|意味|
|:--:|:--:|:--:|:--:|:--:|
|0|0|0|0|A register|
|0|0|0|1|B register|
|0|0|1|0|C register|
|0|0|1|1|D register|
|0|1|0|0|E register|
|0|1|0|1|BP register|
|0|1|1|0|SP register|
|0|1|1|1|オペランド間接アドレッシング|
|1|0|0|0|Input_port|
|1|0|0|1|Output_port|
|1|0|1|0|MEMADDR register|
|1|0|1|1|MEMVAL register|
|1|1|1|1|本領域が使用されていないことを表す|
|X|X|X|X|禁止|

FuncInput2
|bit20|bit19|bit18|bit17|意味|
|:--:|:--:|:--:|:--:|:--:|
|0|0|0|0|A register|
|0|0|0|1|B register|
|0|0|1|0|C register|
|0|0|1|1|D register|
|0|1|0|0|E register|
|0|1|0|1|BP register|
|0|1|1|0|SP register|
|0|1|1|1|オペランド間接アドレッシング|
|1|0|0|0|Input_port|
|1|0|0|1|MEMADDR register|
|1|0|1|0|MEMVAL register|
|1|1|1|1|本領域が使用されていないことを表す|
|X|X|X|X|禁止|


## 凡例
1. opd:オペランドを表します。
2. r:A,B,C,D,E,SP,BPのいずれか一つを表します。r1,r2のように添え字をつけて区別することがあります。
3. FuncInput1=r1 FuncInput1の値はr1レジスタに対応する値です。r1がCレジスタである場合、FuncInput1は``0x10``になります。

本ドキュメントで示すアセンブリ言語はすべてIntel記法を採用しています。例えば``mov a,5``はAレジスタに5をセットすることを意味します。特記ない限り、以下で示す命令はすべてシングル命令です。
## データ転送系命令
### mov
　レジスタからレジスタ、メモリからレジスタへの読み込み、レジスタからメモリへの書き込みを行う命令です。メモリからメモリの書き込みには対応していません。
　mov命令のファンクションコードは``0x08``です。

|ニーモック|FuncInput1|FuncInput2|オペランド|動作
|:--:|:--:|:--:|:--:|:--:|
|mov r1,r2|r1|r2|0x0000|r2レジスタの値をr1レジスタに転送します|
|mov memaddr,r1|memaddr|r1|0x0000|r1レジスタの値をmemaddrレジスタに転送します|
|mov r1,opd|r1|0xf|opd|opdをr1レジスタに転送します
|mov r1,[memaddr+opd]|r1|memaddr|opd|[memaddrレジスタの値にopdを加算した値]番地にあるメモリの値をr1レジスタに転送します※1|
|mov memval,[memaddr+opd]|r1|memaddr|opd|[memaddrレジスタの値にopdを加算した値]番地にあるメモリの値をmemvalレジスタに転送します※1|
|mov memval,r1|memval|r1|0x0000|r1レジスタの値をmemvalレジスタに転送します|
|mov [memaddr+opd],memval|memaddr|memval|opd|memvalレジスタの値を[memaddrレジスタの値にopdを加算した値]番地にあるメモリに転送します。※2|
|mov memval,opd|memval|0xf|opd|opdをmemvalレジスタに転送します|
|mov memaddr,opd|memaddr|0xf|opd|opdをmemaddrレジスタに転送します|

※1　番地指定としてMEMADDRレジスタ以外のレジスタは指定できません。
※2　番地指定としてMEMADDRレジスタ以外は指定できず、なおかつメモリに書き込む値としてMEMVALレジスタ以外は指定できません。

マルチ命令として以下の命令が提供されています。
|ニーモック|動作
|:--:|:--:|
|mov r1,[r2+opd]|[r2レジスタの値にopdを加算した値]番地にあるメモリの値をr1レジスタに転送します|
|mov [r1+opd],r2|r2レジスタの値を[r1レジスタの値にopdを加算した値]番地にあるメモリに転送します。|
|mov [r1+opd1],opd2|opd2を[r1レジスタの値にopd1を加算した値]番地にあるメモリに転送します。|

## 条件付きジャンプ命令
　ゼロフラグ、符号フラグの値によって指定のアドレスにジャンプするかが決まる命令群です。以下の命令は**cmp命令もしくはFLAGSレジスタを書き換える命令の直後に実行してください。** でなければ、結果が条件付きジャンプ命令に正常に伝達されません。以下``k``,``l``とは``cmp k,l``中にある``k``,``l``を指すものとします。

|ニーモック|ファンクションコード|FuncInput1|FuncInput2|オペランド|動作
|:--:|:--:|:--:|:--:|:--:|:--:|
|jnc opd|0xb|0xf|0xf|opd|キャリーフラグが0のときopd番地にジャンプする
|jc opd|0xc|0xf|0xf|opd|キャリーフラグが1のときopd番地にジャンプする
|jnz opd|0x0d|0xf|0xf|opd|ゼロフラグが0のときopd番地にジャンプする
|jz opd|0x0e|0xf|0xf|opd|ゼロフラグが1のときopd番地にジャンプする
|jns opd|0x0f|0xf|0xf|opd|符号フラグが0のときopd番地にジャンプする
|js opd|0x10|0xf|0xf|opd|符号フラグが1のときopd番地にジャンプする
|jnc r1|0xb|0xf|0xf|0x0000|キャリーフラグが0のとき[r1レジスタの値]番地にジャンプする
|jc r1|0xc|0xf|0xf|0x0000|キャリーフラグが1のとき[r1レジスタの値]番地にジャンプする
|jnz r1|0x0d|r1|0xf|0x0000|ゼロフラグが0のとき[r1レジスタの値]番地にジャンプする
|jz r1|0x0e|r1|0xf|0x0000|ゼロフラグが1のとき[r1レジスタの値]番地にジャンプする
|jns r1|0x0f|r1|0xf|0x0000|符号フラグが0のとき[r1レジスタの値]番地にジャンプする
|js r1|0x10|r1|0xf|0x0000|符号フラグが1のとき[r1レジスタの値]番地にジャンプする
|je opd|0x16|0xf|0xf|opd|``k==l``のときopd番地にジャンプする
|jne opd|0x17|0xf|0xf|opd|``k!=l``のときopd番地にジャンプする
|ja opd|0x18|0xf|0xf|opd|``k>l``のときopd番地にジャンプする
|jae opd|0x19|0xf|0xf|opd|``k>=l``のときopd番地にジャンプする
|jl opd|0x1a|0xf|0xf|opd|``k<l``のときopd番地にジャンプする
|jle opd|0x1b|0xf|0xf|opd|``k<=l``のときopd番地にジャンプする
|je r1|0x16|r1|0xf|0x0000|``k==l``のとき[r1レジスタの値]番地にジャンプする
|jne r1|0x17|r1|0xf|0x0000|``k!=l``のとき[r1レジスタの値]番地にジャンプする
|ja r1|0x18|r1|0xf|0x0000|``k>l``のとき[r1レジスタの値]番地にジャンプする
|jae r1|0x19|r1|0xf|0x0000|``k>=l``のとき[r1レジスタの値]番地にジャンプする
|jl r1|0x1a|r1|0xf|0x0000|``k<l``のとき[r1レジスタの値]番地にジャンプする

## スタック操作系命令
### push
　RAMのスタック領域にデータをプッシュします。スタック領域はメモリの上位アドレスから下位アドレスに伸びる仕様となっています。

　push命令はマルチ命令であり、以下のコードと同等です。

push opd
```nasm
sub sp,1
mov memval,{opd}
mov memaddr,sp
mov [memaddr+0],memval
```
push r1
```nasm
sub sp,1
mov memval,{r1}
mov memaddr,sp
mov [memaddr+0],memval
```
|ニーモック|動作
|:--:|:--:|
|push opd|opdをスタック領域にプッシュする|
|push r1|r1レジスタの値をスタック領域にプッシュする|

### pop
　RAMのスタック領域からデータをポップします。
　pop命令はマルチ命令であり、以下のコードと同等です。
```nasm
mov memaddr,sp
mov {r1},[memaddr+0]
add sp,1
```
|ニーモック|動作
|:--:|:--:|
|pop r1|スタック領域からポップした値をr1レジスタに格納する|

## サブルーチン系命令
### call
　指定した番地に書かれているプログラムを呼び出します。call命令はジャンプ命令と異なり、ret命令でコール元へ戻ることができます。

　call命令はマルチ命令であり、以下のコードと同等です。

```nasm
lpc
add memval,0xa
; push memval
sub sp,1
mov memaddr,sp
mov [memaddr+0],memval
jmp {opd}
```
|ニーモック|動作
|:--:|:--:|
|call opd|opd番地に格納されたプログラムを呼び出す|
### ret
　call命令の呼び出し元に制御を戻します。一般的な高水準言語で用いられているreturn文と役割は同じです。
　ret命令はマルチ命令であり、以下のコードと同等です。
```nasm
; pop memval
mov memaddr,sp
mov memval,[memaddr+0]
add sp,1
jmp memval
```
|ニーモック|動作
|:--:|:--:|
|ret|call命令の呼び出し元に戻る|
## 無効命令
　オペコード``0xff``はシステムによって予約されています。将来にわたってこのオペコードが意味のある命令として使われることはなく、またユーザーが意味のあるオペコードとして使用してもいけません。本オペコードがシステムに入力された場合の動作は未定義動作です。
## 未定義命令
　本マニュアルで規定していない命令はすべて未定義命令です。未定義命令が入力された場合の動作は未定義動作となります。
